# 测试计划

> **版本**: v1.0 | **日期**: 2026-07-06 | **负责人**: 组长

---

## 一、测试策略

采用**分层测试**策略，覆盖单元测试 → 集成测试 → 接口测试 → 前端E2E测试。

| 测试层级 | 工具 | 责任人 |
|----------|------|--------|
| 单元测试 | JUnit 5 + Mockito | 后端开发 + 组长审核 |
| Mapper 测试 | MyBatis-Plus + H2 | 后端开发 |
| 接口测试 | Postman / Knife4j | 组长 |
| 前端E2E | 手动测试 / Playwright（可选） | 前端开发 + 组长验收 |

---

## 二、测试环境

| 环境 | 说明 |
|------|------|
| 开发环境 | 本地 MySQL + 本地前端 (:3000) + 本地后端 (:8080) |
| 测试数据库 | H2 内存数据库（单元测试） / MySQL 测试库（集成测试） |

## 三、单元测试用例模板

### 3.1 UserService 测试用例

| 用例ID | 测试场景 | 输入 | 预期结果 |
|--------|----------|------|----------|
| UT-USER-001 | 正常注册 | 新用户名+密码 | 注册成功，返回用户信息 |
| UT-USER-002 | 重复用户名注册 | 已存在用户名 | 抛出 BusinessException "用户名已存在" |
| UT-USER-003 | 正确登录 | 正确用户名+密码 | 登录成功，返回用户信息 |
| UT-USER-004 | 错误密码登录 | 正确用户名+错误密码 | 抛出 BusinessException "用户名或密码错误" |
| UT-USER-005 | 禁用账户登录 | 已禁用账户 | 抛出 BusinessException "账户已被禁用" |

### 3.2 BorrowService 测试用例

| 用例ID | 测试场景 | 输入 | 预期结果 |
|--------|----------|------|----------|
| UT-BORROW-001 | 正常借阅 | 有效用户+有库存图书 | 创建借阅记录，库存-1，用户借阅数+1 |
| UT-BORROW-002 | 超量借阅 | 用户已达最大借阅数 | 抛出异常 "已达到最大借阅数量" |
| UT-BORROW-003 | 缺货借阅 | 库存为0的图书 | 抛出异常 "图书库存不足" |
| UT-BORROW-004 | 正常归还 | 借阅中的记录 | 状态变更为 RETURNED，库存+1 |
| UT-BORROW-005 | 逾期归还 | 超过应还日期 | 状态变更为 OVERDUE，计算逾期罚款 |
| UT-BORROW-006 | 正常续借 | 借阅中+未超过续借次数 | due_date +15天，renew_count+1 |
| UT-BORROW-007 | 超次续借 | 已续借2次 | 抛出异常 "已达到最大续借次数" |

### 3.3 OrderService 测试用例

| 用例ID | 测试场景 | 输入 | 预期结果 |
|--------|----------|------|----------|
| UT-ORDER-001 | 正常创建订单 | 有效图书+数量 | 创建订单+订单项，扣减库存 |
| UT-ORDER-002 | 库存不足创建 | 数量>库存 | 抛出异常 "库存不足" |
| UT-ORDER-003 | 正常支付 | 待付款订单 | 状态变为 PAID→PROCESSING |
| UT-ORDER-004 | 取消订单 | 待付款订单 | 状态变为 CANCELLED，恢复库存 |
| UT-ORDER-005 | 已支付取消失败 | 已支付订单 | 抛出异常 "只能取消待付款订单" |
| UT-ORDER-006 | 正常发货 | 处理中订单 | 状态变为 SHIPPED |

## 四、接口测试清单

| 接口 | 方法 | 测试要点 |
|------|------|----------|
| `/api/user/register` | POST | 参数校验、重复注册、密码加密 |
| `/api/user/login` | POST | 正常登录、错误密码、禁用账户 |
| `/api/book/search` | GET | 关键词搜索、分类筛选、分页、空结果 |
| `/api/book/{id}` | GET | 正常查询、不存在的ID、已下架图书 |
| `/api/borrow` | POST | 正常借阅、库存不足、超量借阅 |
| `/api/borrow/return/{id}` | PUT | 正常归还、重复归还、逾期归还 |
| `/api/borrow/renew/{id}` | PUT | 正常续借、超次数续借 |
| `/api/order` | POST | 创建订单、空购物车、库存校验 |
| `/api/order/pay/{id}` | PUT | 支付、重复支付 |
| `/api/order/cancel/{id}` | PUT | 取消、已支付取消 |
